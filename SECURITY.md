# Política de Segurança (Security Policy)

## Versões Suportadas

Atualmente, apenas a versão em desenvolvimento ativo na branch `main` recebe correções de segurança. À medida que o projeto OpenTaipa atingir versões estáveis (v1.0.0+), definiremos ciclos formais de suporte de longo prazo (LTS) aqui.

| Versão | Suportada |
| :--- | :--- |
| **Edge / Main** | ✅ Sim |
| **< 1.0.0** | ❌ Não (Versões Alfa/Beta de desenvolvimento) |

## Como Reportar uma Vulnerabilidade

> [!IMPORTANT]
> **Por favor, não abra Issues públicas para relatar falhas de segurança.** 
> Relatar bugs de segurança publicamente expõe os usuários do OpenTaipa a riscos antes que uma correção esteja disponível.

Para reportar uma vulnerabilidade de forma segura e confidencial, utilize uma das seguintes vias:

1. **GitHub Private Vulnerability Reporting (Recomendado):**
   * Vá para a aba **Security** (Segurança) do repositório correspondente no GitHub.
   * Clique em **Advisories** e depois em **Report a vulnerability** (Reportar uma vulnerabilidade).
   * Este método mantém a comunicação segura e criptografada diretamente entre você e os mantenedores do projeto dentro do próprio GitHub.

2. **Por E-mail:**
   * Envie um e-mail detalhado para **fsp.passos@gmail.com** com o assunto `[SECURITY] Vulnerabilidade no OpenTaipa`.
   * Se possível, inclua um passo a passo para reproduzir a falha (PoC) e detalhes sobre o ambiente testado.

## Nosso Compromisso de Resposta

* **Triagem Inicial:** Confirmaremos o recebimento do seu reporte em até **48 horas**.
* **Correção e Divulgação:** Trabalharemos em conjunto com você para criar uma correção rápida. Manteremos você informado sobre o progresso e daremos os devidos créditos pela descoberta no momento do lançamento da nova versão corretiva, a menos que você prefira permanecer anônimo.
